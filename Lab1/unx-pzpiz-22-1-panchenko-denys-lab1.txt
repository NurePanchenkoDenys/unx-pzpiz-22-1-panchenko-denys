2 ЗАВДАННЯ

     Розробити Bash-скрипт, який реалізує взаємодію двох процесів (батьківського та дочірнього) з використанням сигналів та механізмів журналювання.

     Вимоги до скрипта:
     1) Обробляти параметри командного рядка: -h (довідка), некоректні параметри - помилка у stderr.
     2) Батьківський процес працює в інтерактивному режимі, приймає команди, надсилає сигнали дочірньому процесу.
     3) Дочірній процес виконується у фоновому режимі, обробляє сигнали від батька, надсилає підтвердження, ігнорує SIGINT.
     4) Використовувати сигнали SIGTERM, SIGUSR1, SIGUSR2, SIGHUP.
     5) Вести локальний журнал у форматі: дата; мітка часу; PID; номер сигналу; назва сигналу; опис.
     6) Записувати ключові події у системний журнал syslog через команду logger.
     7) Забезпечити локалізацію повідомлень (українська при uk_UA, англійська - інакше).
     8) Уникати зомбі-процесів, коректно обробляти завершення.


3 ОПИС ВИКОНАНОЇ РОБОТИ

3.1 Загальна архітектура скрипта

     Скрипт реалізує модель взаємодії двох процесів: батьківського (інтерактивного) та дочірнього (фонового). Батьківський процес приймає команди від користувача та надсилає сигнали дочірньому. Дочірній процес обробляє сигнали та надсилає підтвердження назад. Обидва процеси ведуть спільний журнал подій.
     Скрипт має модульну структуру з окремими функціями для кожної задачі: локалізація, журналювання, обробка параметрів, керування процесами.

3.2 Обробка параметрів командного рядка

     Реалізовано функцію parse_args з конструкцією case. Підтримуються параметри -h та --help для виведення довідки. Невідомі параметри та зайві аргументи відхиляються з повідомленням у stderr та ненульовим кодом виходу.

3.3 Локалізація повідомлень

     Функція is_ukrainian перевіряє змінні середовища LC_MESSAGES та LANG. При українській локалізації (uk_UA) повідомлення виводяться українською, в інших випадках - англійською. Це стосується довідки, повідомлень про помилки та інформаційних повідомлень.

3.4 Дочірній процес

     Дочірній процес запускається у фоновому режимі за допомогою оператора &. Він встановлює обробники сигналів через команду trap:

     SIGUSR1 - при отриманні записує подію у журнал та надсилає підтвердження батьківському процесу через kill -SIGUSR1 $PPID.
     SIGUSR2 - аналогічна логіка з надсиланням підтвердження SIGUSR2.
     SIGHUP - записує подію про перезавантаження конфігурації.
     SIGTERM - коректне завершення з записом у обидва журнали та виходом.
     SIGINT - ігнорується (trap з порожнім рядком), щоб Ctrl+C не завершував дочірній процес.

     Основний цикл дочірнього процесу - нескінченний цикл while true зі sleep 1, що забезпечує очікування сигналів.

3.5 Батьківський процес

     Батьківський процес працює в інтерактивному режимі. Після запуску дочірнього процесу він входить у цикл, де зчитує команди користувача через read та обробляє їх конструкцією case:

     status - перевіряє стан дочірнього процесу через kill -0.
     send USR1/USR2/HUP - надсилає відповідний сигнал дочірньому процесу.
     stop - надсилає SIGTERM дочірньому та очікує його завершення через wait.
     quit - викликає функцію cleanup_and_exit для коректного завершення.
     help - виводить довідку.

3.6 Коректне завершення та запобігання зомбі-процесам

     Функція cleanup_and_exit перевіряє, чи активний дочірній процес. Якщо так - надсилає SIGTERM та виконує wait, що очікує завершення процесу. Це запобігає появі зомбі-процесів. Функція також обробляє ситуації натискання Ctrl+C та отримання SIGTERM батьківським процесом.

3.7 Журналювання

     Реалізовано два рівні журналювання:

     Локальний журнал - файл ~/log/...lab1.log. Кожен запис містить дату (у форматі RFC 2822 з LC_ALL=C), мітку часу Unix, PID процесу, номер сигналу, назву сигналу та текстовий опис. Роздільник - крапка з комою.

     Системний журнал - запис через команду logger з тегом, що дорівнює імені скрипта. Фіксуються: запуск скрипта, створення дочірнього процесу, завершення процесів.

     Батьківський процес працює в інтерактивному режимі та приймає команди користувача, дочірній виконується у фоновому режимі та обробляє сигнали SIGUSR1, SIGUSR2, SIGHUP та SIGTERM з підтвердженням отримання.

     Набуто практичні навички роботи з процесами, сигналами, міжпроцесною взаємодією та журналюванням подій в операційних системах Unix.

 93  # Функція: запис у локальний журнал
 94  # Формат: <дата>; <timestamp>; <PID>; <signal_number>; <signal_name>; <опис>

 96  log_event() {
 97      local pid="$1"
 98      local sig_num="$2"
 99      local sig_name="$3"
100      local description="$4"
101      local date_str
102      date_str="$(LC_ALL=C date "+%a, %d %b %Y %H:%M:%S %z")"
103      local timestamp
104      timestamp="$(date +%s)"
105  
106      echo "${date_str}; ${timestamp}; ${pid}; ${sig_num}; ${sig_name}; ${description}" >> "$LOG_FILE"

114  }

116  # =============================================================================
117  # Функція: створення каталогу для журналу
118  # =============================================================================
119  ensure_log_dir() {
120      if [ ! -d "$LOG_DIR" ]; then
121          if ! mkdir -p "$LOG_DIR" 2>/dev/null; then
122              error_msg "$(is_ukrainian && echo "неможливо створити каталог: $LOG_DIR" || echo "cannot create directory: $LOG_DIR")"

124          fi
125      fi

128  # =============================================================================
129  # Функція: перевірка існування дочірнього процесу
130  # =============================================================================
131  is_child_alive() {
132      if [ -n "$CHILD_PID" ] && kill -0 "$CHILD_PID" 2>/dev/null; then
133          return 0
134      fi
135      return 1
136  }

138  # =============================================================================
139  # Дочірній процес
140  # =============================================================================
141  child_process() {
142      local my_pid=$$
143  
144      # Обробник SIGUSR1 - повідомлення від батька
145      trap '{
146          log_event "'"$my_pid"'" "10" "SIGUSR1" "Child: received SIGUSR1 from parent"
147          info_msg "[Дочірній] Отримано SIGUSR1. Надсилаю підтвердження." "[Child] Received SIGUSR1. Sending confirmation."
148          kill -SIGUSR1 "$PPID" 2>/dev/null
149      }' SIGUSR1
150  
151      # Обробник SIGUSR2 - додаткова дія
152      trap '{
153          log_event "'"$my_pid"'" "12" "SIGUSR2" "Child: received SIGUSR2 from parent"
154          info_msg "[Дочірній] Отримано SIGUSR2. Виконую додаткову дію." "[Child] Received SIGUSR2. Performing additional action."
155          kill -SIGUSR2 "$PPID" 2>/dev/null
156      }' SIGUSR2
157  
158      # Обробник SIGHUP - перезавантаження конфігурації
159      trap '{
160          log_event "'"$my_pid"'" "1" "SIGHUP" "Child: received SIGHUP, reloading"
161          info_msg "[Дочірній] Отримано SIGHUP. Перезавантаження." "[Child] Received SIGHUP. Reloading."
162      }' SIGHUP
163  
164      # Обробник SIGTERM - коректне завершення
165      trap '{
166          log_event "'"$my_pid"'" "15" "SIGTERM" "Child: received SIGTERM, exiting"
167          log_syslog "Child process $my_pid terminated by SIGTERM"
168          info_msg "[Дочірній] Отримано SIGTERM. Завершення роботи." "[Child] Received SIGTERM. Exiting."
169          exit 0
170      }' SIGTERM
171  
172      # Ігнорування SIGINT (Ctrl+C) - дочірній не завершується від Ctrl+C
173      trap '' SIGINT
174  
175      log_event "$my_pid" "0" "NONE" "Child process started"
176      log_syslog "Child process started with PID $my_pid"
177      info_msg "[Дочірній] Процес запущено (PID: $my_pid)" "[Child] Process started (PID: $my_pid)"
178  
179      # Основний цикл дочірнього процесу
180      while true; do
181          sleep 1
182      done
183  }
184  
185  # =============================================================================
186  # Обробники сигналів батьківського процесу
187  # =============================================================================
188  
189  # Підтвердження від дочірнього (SIGUSR1)
190  handle_parent_sigusr1() {
191      log_event "$$" "10" "SIGUSR1" "Parent: received confirmation SIGUSR1 from child"
192      info_msg "[Батьківський] Отримано підтвердження SIGUSR1 від дочірнього." "[Parent] Received SIGUSR1 confirmation from child."
193  }
194  
195  # Підтвердження від дочірнього (SIGUSR2)
196  handle_parent_sigusr2() {
197      log_event "$$" "12" "SIGUSR2" "Parent: received confirmation SIGUSR2 from child"
198      info_msg "[Батьківський] Отримано підтвердження SIGUSR2 від дочірнього." "[Parent] Received SIGUSR2 confirmation from child."
199  }
200  
201  # Обробка SIGINT (Ctrl+C) - коректне завершення
202  handle_parent_sigint() {
203      echo ""
204      info_msg "[Батьківський] Отримано SIGINT (Ctrl+C). Завершення..." "[Parent] Received SIGINT (Ctrl+C). Shutting down..."
205      log_event "$$" "2" "SIGINT" "Parent: received SIGINT, shutting down"
206      cleanup_and_exit
207  }
208  
209  # Обробка SIGTERM
210  handle_parent_sigterm() {
211      info_msg "[Батьківський] Отримано SIGTERM. Завершення..." "[Parent] Received SIGTERM. Shutting down..."
212      log_event "$$" "15" "SIGTERM" "Parent: received SIGTERM, shutting down"
213      cleanup_and_exit
214  }
215  
216  # =============================================================================
217  # Функція: коректне завершення з очищенням
218  # =============================================================================
219  cleanup_and_exit() {
220      if is_child_alive; then
221          info_msg "[Батьківський] Зупиняю дочірній процес (PID: $CHILD_PID)..." "[Parent] Stopping child process (PID: $CHILD_PID)..."
222          kill -SIGTERM "$CHILD_PID" 2>/dev/null
223          # Очікування завершення дочірнього процесу
224          wait "$CHILD_PID" 2>/dev/null
225      fi
226      log_event "$$" "0" "NONE" "Parent process exiting"
227      log_syslog "Parent process $$ exiting"
228      info_msg "[Батьківський] Роботу завершено." "[Parent] Exiting."
229      exit 0
230  }

232  # =============================================================================
233  # Функція: надсилання сигналу дочірньому процесу
234  # =============================================================================
235  send_signal_to_child() {
236      local signal="$1"
237      local sig_num="$2"
238      local sig_name="$3"
239  
240      if ! is_child_alive; then
241          error_msg "$(is_ukrainian && echo "дочірній процес не існує" || echo "child process does not exist")"
242          return 1
243      fi
244  
245      kill "-${signal}" "$CHILD_PID" 2>/dev/null
246      if [ $? -eq 0 ]; then
247          log_event "$$" "$sig_num" "$sig_name" "Parent: sent $sig_name to child $CHILD_PID"
248          info_msg "[Батьківський] Надіслано $sig_name дочірньому (PID: $CHILD_PID)" "[Parent] Sent $sig_name to child (PID: $CHILD_PID)"
249      else
250          error_msg "$(is_ukrainian && echo "не вдалося надіслати сигнал" || echo "failed to send signal")"
251      fi
252  }
